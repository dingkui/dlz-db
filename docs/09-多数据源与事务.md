# DLZ-DB 帮助文档 - 多数据源与事务

## 9.1 配置

DLZ-DB 支持动态数据源，通过代码方式添加和切换数据源。

### 添加数据源

```java
// 添加从库数据源
DataSourceProperty slaveProperty = new DataSourceProperty();
slaveProperty.setName("slave");
slaveProperty.setUrl("jdbc:mysql://slave-host:3306/db_slave");
slaveProperty.setUsername("root");
slaveProperty.setPassword("slave123");
slaveProperty.setDriverClassName("com.mysql.cj.jdbc.Driver");

DB.Dynamic.setDataSource(slaveProperty);

// 添加报表库数据源
DataSourceProperty reportProperty = new DataSourceProperty();
reportProperty.setName("report");
reportProperty.setUrl("jdbc:mysql://report-host:3306/db_report");
reportProperty.setUsername("root");
reportProperty.setPassword("report123");
reportProperty.setDriverClassName("com.mysql.cj.jdbc.Driver");

DB.Dynamic.setDataSource(reportProperty);
```

### 设置默认数据源

默认数据源由 Spring Boot 自动配置（注册到名为 `default` 的数据源）。如需通过代码替换默认数据源：

```java
DB.Dynamic.setDefaultDataSource(dataSource);
```

### 删除数据源

```java
DB.Dynamic.removeDataSource("slave");
```

## 9.2 数据源切换与事务

DLZ-DB 将“数据源切换”与“事务管理”拆分为两个独立的入口，职责清晰、可自由组合：

| 入口 | 职责 |
| --- | --- |
| `DB.Dynamic.use(name, ...)` | 仅切换数据源，**不开启事务** |
| `DB.Tx.run(() -> ...)` | 在当前数据源上**开启事务**（未切换时使用默认数据源） |
| `DB.Tx.run(name, () -> ...)` | 切换数据源 **+ 开启事务**（语法糖） |

### 切换数据源（无事务）

```java
// 使用默认数据源（无 use 包裹）
DB.Pojo.select(User.class).queryBeanList();

// 切换到从库查询
List<User> users = DB.Dynamic.use("slave", () ->
    DB.Pojo.select(User.class)
        .eq(User::getStatus, 1)
        .orderByDesc(User::getCreateTime)
        .queryBeanList()
);

// 切换到报表库
List<ResultMap> report = DB.Dynamic.use("report", () ->
    DB.Jdbc.select("SELECT * FROM daily_report").queryList()
);

// Runnable 重载（无返回值）
DB.Dynamic.use("slave", () -> DB.Pojo.insert(user));
```

### 事务控制

`DB.Tx` 基于 Spring `TransactionSynchronizationManager` 实现事务，无需手动管理 Connection。执行流程：

1. 从数据源获取 Connection 并 `setAutoCommit(false)`
2. 通过 `ConnectionHolder` 绑定到 Spring 事务上下文（JdbcTemplate 自动复用同一连接）
3. 代码块正常返回 → `commit()`；抛出异常 → `rollback()`
4. finally 解绑资源、恢复 autoCommit、关闭连接

异常时会自动回滚，并包装为 `DbException(code=1003 或 1006)` 抛出。

#### 默认数据源事务

```java
// 使用 default 数据源执行事务
DB.Tx.run(() -> {
    DB.Pojo.insert(user);
    DB.Pojo.insert(userLog);
});

// 带返回值的重载
Long id = DB.Tx.run(() -> {
    DB.Pojo.insert(user);
    return user.getId();
});
```

#### 指定数据源事务

```java
// 语法糖：一行完成 “切换 + 事务”
DB.Tx.run("slave", () -> {
    DB.Pojo.insert(user);
    DB.Pojo.insert(userLog);
});

// 等价写法（嵌套）：
DB.Dynamic.use("slave", () -> DB.Tx.run(() -> {
    DB.Pojo.insert(user);
    DB.Pojo.insert(userLog);
}));
```

#### 与 Spring `@Transactional` 的关系

- Spring 的 `@Transactional` 仅管理绑定到 Spring 容器的默认 DataSource，无法感知 `DB.Dynamic.use(name, ...)` 切换后的数据源。
- 在 `@Transactional` 方法中调用 `DB.Dynamic.use(otherName, ...)`，**外层 Spring 事务管不到内部操作**。
- 推荐做法：跨数据源场景下使用 `DB.Tx.run(name, () -> ...)` 代替 `@Transactional`。

```java
@Service
public class UserService {

    // 默认数据源：可使用 Spring 事务
    @Transactional
    public void createUser(User user) {
        DB.Pojo.insert(user);
        DB.Pojo.insert(userLog);
    }

    // 切换到 slave：使用框架事务
    public void createOnSlave(User user) {
        DB.Tx.run("slave", () -> {
            DB.Pojo.insert(user);
            DB.Pojo.insert(userLog);
        });
    }
}
```

#### 跨数据源事务

DLZ-DB 不支持跨数据源事务（一次 `Tx.run` 只绑定一个 Connection）。如需跨数据源事务，请使用分布式事务框架（如 Seata）。

### 读写分离建议

```java
@Service
public class UserService {

    // 写操作 → 主库（带事务）
    public void save(User user) {
        DB.Tx.run("master", () -> DB.Pojo.insert(user));
    }

    // 读操作 → 从库（无事务）
    public User getById(Long id) {
        return DB.Dynamic.use("slave", () ->
            DB.Pojo.select(User.class)
                .eq(User::getId, id)
                .queryBean()
        );
    }

    // 需要读取最新数据 → 主库
    public User getByIdFromMaster(Long id) {
        return DB.Dynamic.use("master", () ->
            DB.Pojo.select(User.class)
                .eq(User::getId, id)
                .queryBean()
        );
    }
}
```
