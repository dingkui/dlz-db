## 修正一：Spring 配置类名错误（影响 8 个文件）

**错误写法**：
```java
import com.dlz.db.spring.config.DlzDbConfig;
extends DlzDbConfig
```

**正确写法**：
```java
import com.dlz.db.spring.config.SpringDlzDbConfig;
extends SpringDlzDbConfig
```

**原因**：源码中 Spring Boot 的配置类是 `SpringDlzDbConfig`（`dlz-db-spring-boot-starter/src/main/java/com/dlz/db/spring/config/SpringDlzDbConfig.java`），不存在名为 `DlzDbConfig` 的类。文档中大量引用了一个不存在的类名，会导致用户编译错误。

**影响范围**：

- `docs/第01章-快速入门/1.1-安装配置-SpringBoot.md`

## 修正二：likeLeft / likeRight 说明写反（影响 3 个文件）

**错误写法**（文档中的注释）：
```java
likeLeft(field, value)    // LIKE 'value%'   右模糊
likeRight(field, value)   // LIKE '%value'   左模糊
```

**正确写法**：
```java
likeLeft(field, value)    // LIKE '%value'   后缀匹配（%在左）
likeRight(field, value)   // LIKE 'value%'   前缀匹配（%在右）
```

**原因**：源码实现（`DbOperateEnum.mk()`）：
```java
case likeLeft:  return paraOne(dbn, "%" + value, null);   // → LIKE '%v'
case likeRight: return paraOne(dbn, value + "%", null);    // → LIKE 'v%'
```
方法名表示 `%` 的位置：`likeLeft` = `%`在左（`'%v'`），`likeRight` = `%`在右（`'v%'`）。

> ⚠️ `ICondAddByLamda.java` 接口的 JavaDoc 注释也存在同样的错误，建议后续一并修正。

**影响范围**：
- `docs/DLZ-DB完整功能清单.md`
- `docs/第02章-基础操作/2.3-条件构造器.md`
- `docs/blog/00-DLZ-DB架构全览-AI参考.md`

## 修正三：`.select()` 方法不存在 → `.columns()`（影响 1 个文件）

**错误写法**：
```java
DB.pojo.selectWrapper(User.class)
    .select(User::getId, User::getName)    // ❌ .select() 不存在
```

**正确写法**：
```java
DB.pojo.selectWrapper(User.class)
    .columns(User::getId, User::getName)   // ✅
```

**原因**：`PojoQuery` 查询列的方法是 `columns()`，不是 `select()`。`select()` 是入口类 `DbPojo` 上的静态工厂方法（如 `DB.pojo.selectWrapper(User.class)`），不是条件构造方法。

**影响范围**：`docs/第02章-基础操作/2.1-查询操作.md`

## 修正五：`.queryStrList(String.class)` / `.queryStr(String.class)` 不存在（影响 1 个文件）

**错误写法**：
```java
.queryStrList(String.class)    // ❌ 标量查询方法不接受 Class 参数
.queryStr(String.class)        // ❌
```

**正确写法**：
```java
.queryStrList()                // ✅ 无参，返回 List<String>
.queryStr()                    // ✅ 无参，返回 String
```

**原因**：`IExecutorQuery` 中标量查询方法（`queryStr()`, `queryStrList()`, `queryLong()`, `queryInt()`, `queryDouble()` 及对应的 List 版本）均**不接受 Class 参数**。带 Class 参数的只有 `queryOne(Class)`, `queryList(Class)`, `queryPage(Class)`。

**影响范围**：`docs/第02章-基础操作/2.1-查询操作.md`

## 修正十：`DB.sequence()` 不存在（影响 2 个文件）

**错误写法**：
```java
DB.sequence("order", 100)    // ❌ DB 类上没有 sequence 方法
```

**正确写法**：
```java
DBHolder.sequence("order", 100)    // ✅ sequence 在 DBHolder 上
```

**原因**：`DB.java` 只有 7 个 `public final static` 字段（Jdbc、Table、Sql、Pojo、Batch、Dynamic、Tx），没有 `sequence` 方法。分布式 ID 生成通过 `DBHolder.sequence()` 调用。

**影响范围**：
- `docs/blog/00-DLZ-DB架构全览-AI参考.md`
- `docs/blog/06-智能号段ID生成零浪费的分布式实践.md`

## 关于 `.page()` 的更正

**初始判断为错误（已撤回）**：最初认为 `.page(pageNum, pageSize)` 是错误的，应该改成 `.page(Page.build(pageNum, pageSize))`。

**实际情况**：`ISqlPage` 接口提供了 default 方法：
```java
default T page(int pageIndex, int size, Order... orders) { ... }
default T page(long current, long size, List<Order> orders) { ... }
```

因此 `.page(1, 10)` 和 `.page(pageNum, pageSize)` 都是正确的写法，两者均可使用。
