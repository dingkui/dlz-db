# AI Quick Reference

> Static entry `DB.`, chain API, no Mapper/XML. Identical under Spring Boot and Solon.

---

## I. Entry Points

| Entry | Use Case | Placeholder |
|-------|----------|-------------|
| `DB.Pojo` | CRUD with Bean (preferred) | Condition builder |
| `DB.Table` | Dynamic table names | Condition builder |
| `DB.Jdbc` | One-time SQL | `?` |
| `DB.Sql` | Preset/complex SQL | `#{key}` |
| `DB.Tx` | Programmatic transaction | — |
| `DB.Batch` | Batch write | — |
| `DB.Dynamic` | Datasource switching | — |

## II. Condition Methods

All methods support three-parameter form `eq(boolean, field, value)`, skip when false.

| Method | SQL |
|--------|-----|
| `eq / ne / gt / ge / lt / le` | `= / <> / > / >= / < / <=` |
| `isNull / isNotNull` | `IS NULL / IS NOT NULL` |
| `in / notIn` | `IN / NOT IN` |
| `between / notBetween` | `BETWEEN / NOT BETWEEN` |
| `like / likeLeft / likeRight / notLike` | LIKE four variants |

`in()` value: `List`, CSV string, or `"sql:subquery"`, **cannot pass single value**.

Nesting: `or(o -> ...)` / `and(a -> ...)`. Custom fragment: `sql("col = #{k}", map)`. Specify columns: `columns(fields...)` (not `select()`).

## III. Return Values

| Method | Return |
|--------|--------|
| `queryBean() / queryBeanList() / queryBeanPage()` | `T / List<T> / Page<T>` |
| `queryOne() / queryList() / queryPage()` | `ResultMap / List / Page` |
| `queryOne(C) / queryList(C) / queryPage(C)` | Specified type `C` |
| `count()` | `int` |
| `queryStr/Int/Long/Double()` | Single column scalar |
| `execute()` | `int` |

Rule: with `Bean` → Bean; without → `ResultMap`; with `(Class)` → specified type. Write operations must end with `.execute()`.

## IV. Transactions

Declarative: Spring Boot `@Transactional`, Solon `@Tran`.

Programmatic (framework-independent):
```java
DB.Tx.run(() -> { ... });                    // Default datasource
DB.Tx.run("slave", () -> { ... });           // Specified datasource
Long id = DB.Tx.run(() -> { insert(); return id; });  // With return value
```

Solon does not support `REQUIRES_NEW` / `NESTED`.

## V. Datasource Switching

```java
DB.Dynamic.use("slave", () -> DB.Pojo.select(...).queryBeanList());
DB.Dynamic.setDataSource(prop);  // Dynamic registration
```

`DB.Dynamic.use()` is independent of `@Transactional` / `@Tran`. Use `DB.Tx.run("slave", ...)` for switching within transactions.

## VI. Hard Constraints

1. No Mapper/DAO/Wrapper classes, use `DB.Pojo.*` directly
2. Placeholder separation: `DB.Jdbc` uses `?`; `DB.Sql` / `sql()` uses `#{key}`
3. `queryOne/List/Page` returns `ResultMap`, use `queryBean` series for Bean
4. Query columns use `columns()` not `select()` (`.columns(User::getId, User::getName)`)
5. LIKE uses `like()` / `likeLeft()` / `likeRight()` / `notLike()`, not just `like()`
6. Insert with auto-increment key: `.insertWithAutoKey()` not `.execute()`
7. Physical delete / query bypass logical delete: `.ignoreLogicDelete(true)`
8. Preset SQL key starts with `"key."`
9. `count()` returns `int`
10. `in()` cannot pass single value
11. Batch operations use `DB.Batch.insert(users)`, not `insertBatch()`

## VII. Entity Conventions

- Table name: CamelCase to snake_case (User.class → user). Custom: `@TableName("t_user")`
- Field: `User::getUserName` → `user_name`. Custom: `@TableField("email_address")`
- Primary key: `@TableId`
- Logical delete: Bean with `isDeleted` auto-enables (query adds `is_deleted = 0`, DELETE converts to UPDATE)

## VIII. Code Templates

```java
// CRUD
User u = DB.Pojo.select(User.class).eq(User::getId, 1).queryBean();
List<User> list = DB.Pojo.select(User.class).like(User::getName, "Zhang").queryBeanList();
Page<User> page = DB.Pojo.select(User.class).eq(User::getStatus, 1).page(1, 10).queryBeanPage();
DB.Pojo.insert(user);
DB.Pojo.updateById(user);
DB.Pojo.delete(User.class).eq(User::getId, 1).execute();

// Batch
DB.Batch.insert(users);

// Native SQL
List<ResultMap> rows = DB.Jdbc.select("SELECT * FROM user WHERE id = ?", 1).queryList();

// Preset SQL
List<User> r = DB.Sql.select("key.user.find").addPara("status", 1).queryList(User.class);

// Transaction
DB.Tx.run(() -> { DB.Pojo.insert(order); DB.Pojo.insert(orderItem); });
```
