# DLZ-DB 代码质量检查 - 快速开始

## ✅ 已完成配置

质量检查插件已经集成到项目的 `pom.xml` 中，包括：

- ✅ JaCoCo (测试覆盖率)
- ✅ SpotBugs + FindSecBugs (静态分析 + 安全检查)
- ✅ Checkstyle (代码规范)
- ✅ PMD (代码质量)

## 🚀 立即开始

### 方式一：快速检查（推荐日常使用）

**适用场景**：提交代码前的快速验证

**Windows:**
```cmd
quick-check.bat
```

**Linux/Mac:**
```bash
chmod +x quick-check.sh
./quick-check.sh
```

**检查内容**：
- 编译项目
- 运行单元测试
- 生成覆盖率报告
- SpotBugs 静态分析

**耗时**：约 1-3 分钟

---

### 方式二：完整检查（推荐 PR 前使用）

**适用场景**：Pull Request 前的全面质量检查

**Windows:**
```cmd
run-quality-checks.bat
```

**Linux/Mac:**
```bash
chmod +x run-quality-checks.sh
./run-quality-checks.sh
```

**检查内容**：
- 编译项目
- 运行单元测试

- SpotBugs 静态分析
- Checkstyle 代码规范
- PMD 代码质量

- 生成综合报告站点

**耗时**：约 3-5 分钟

## 🔧 调整质量阈值

编辑根目录的 `pom.xml`，找到以下配置：

```xml
<properties>
    <!-- 质量阈值 -->
    <jacoco.line.coverage>0.70</jacoco.line.coverage>      <!-- 行覆盖率 70% -->
    <jacoco.branch.coverage>0.60</jacoco.branch.coverage>  <!-- 分支覆盖率 60% -->
</properties>
```

根据项目实际情况调整数值（0.0 到 1.0 之间）。

## ❓ 常见问题

### Q1: 首次运行很慢？

**A**: 首次运行需要下载 Maven 插件和依赖，这是正常的。后续运行会快很多。

### Q2: 测试失败怎么办？

**A**: 
1. 查看 `target/surefire-reports/` 目录下的测试报告
2. 修复失败的测试
3. 重新运行 `quick-check.bat`

### Q3: SpotBugs 报告很多问题？

**A**: 
1. 优先修复 "High" 和 "Medium" 级别的问题
2. "Low" 级别的问题可以逐步改进
3. 如果是误报，可以使用 `@SuppressFBWarnings` 注解

### Q4: Checkstyle 警告太多？

**A**: 
- 当前配置为 `failsOnError=false`，不会阻止构建
- 可以逐步修复，或自定义 Checkstyle 规则
- 参考 `QUALITY-TESTING-GUIDE.md` 了解如何自定义规则

### Q5: 如何在 IDE 中查看覆盖率？

**A**: 
- **IntelliJ IDEA**: 右键点击测试类 → "Run with Coverage"
- **Eclipse**: 安装 EclEmma 插件
- 或者直接在浏览器中打开 `target/site/jacoco-aggregate/index.html`

## 📝 工作流建议

### 日常开发流程

```
1. 编写代码
2. 编写测试
3. 运行 quick-check.bat (快速验证)
4. 修复问题
5. 提交代码
```

### Pull Request 流程

```
1. 完成功能开发
2. 运行 run-quality-checks.bat (完整检查)
3. 查看所有报告
4. 修复发现的问题
5. 更新 quality-metrics-template.json (可选)
6. 创建 Pull Request
```

### 定期维护

```
每周：
- 运行完整质量检查
- 关注新增的问题
- 逐步提高质量阈值

每月：
- 运行依赖安全扫描 (mvn dependency-check:check)
- 更新有漏洞的依赖
```
