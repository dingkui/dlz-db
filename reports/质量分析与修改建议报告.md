### 2.3 关键安全漏洞说明

#### 🔴 SQL 注入（最高优先级）

**`SpringSqlExecutorAdapter.java`**
- FindSecBugs 规则：`SECSQLISPRJDBC`（Spring JDBC SQL Injection）
- CWE-89：SQL 命令中特殊元素的不当中性化
- 触发场景：如果 SQL 字符串通过外部参数动态拼接后传给 `JdbcTemplate.query/execute`

**`SolonSqlExecutorAdapter.java`**
- FindSecBugs 规则：`SECSQLIJDBC`（JDBC SQL Injection）+ `SQL_NONCONSTANT_STRING_PASSED_TO_EXECUTE`
- 触发场景：非常量字符串传给 `Statement.execute` 或 `addBatch`

**修复方向**：
1. 确认所有外部输入的 SQL 参数是否通过 `PreparedStatement` 绑定（`?` 占位符）
2. 如果必须动态拼接 SQL（如按条件增减 WHERE 子句），确保拼接部分**不是用户输入**
3. 对排序字段、表名等无法参数化的位置，使用**白名单校验**

#### 🟡 可变对象暴露（高优先级）

`EI_EXPOSE_REP` / `EI_EXPOSE_REP2` 在 core、spring、solon 三个模块广泛出现。

**影响**：外部代码拿到内部可变对象引用后可直接修改，破坏封装。

**修复方向**：
- 返回/存储前做防御性拷贝：`new ArrayList<>(list)`、`Collections.unmodifiableList(list)`
- 对于数组：`array.clone()`

#### 🟡 静态字段线程安全（中优先级）

`LI_LAZY_INIT_STATIC` + `ST_WRITE_TO_STATIC_FROM_INSTANCE_METHOD`

**影响**：多线程环境下可能看到未完全初始化的对象，或发生竞态条件。

**修复方向**：
- 懒初始化加 `synchronized` 或改用 `volatile` + DCL
- 实例方法不应修改静态状态，考虑改为实例字段或同步块

#### 🟡 资源泄漏（中优先级）

`OBL_UNSATISFIED_OBLIGATION_EXCEPTION_EDGE` — 异常时流/资源未关闭。

**修复方向**：所有 `Connection`、`Statement`、`ResultSet` 使用 try-with-resources 或 try-finally。
