# 如何运行代码质量测试

## 🎯 快速开始

### 方式 1：从项目根目录

```cmd
quality-test.bat
```

### 方式 2：从 reports 目录

```cmd
cd reports
test-all.bat
```

---

## 📋 单项测试

在 `reports` 目录下运行：

```cmd
cd reports

# 1. 测试覆盖率（推荐优先运行）
test-jacoco.bat

# 2. 静态分析 + 安全检查
test-spotbugs.bat

# 3. 代码规范检查
test-checkstyle.bat

# 4. 代码质量分析
test-pmd.bat

# 5. 依赖安全扫描（耗时较长，可选）
test-owasp.bat
```

---

## 📊 查看报告

### 方式 1：使用报告查看器（推荐）

```cmd
cd reports
view-reports.bat
```

会显示交互式菜单，选择要查看的报告。

### 方式 2：直接打开文件

```cmd
# JaCoCo 覆盖率
start target\reports\jacoco-aggregate\index.html

# SpotBugs 静态分析
start target\reports\spotbugs\spotbugsXml.html

# Checkstyle 代码规范
start target\site\checkstyle.html

# PMD 代码质量
start target\site\pmd.html

# CPD 重复代码
start target\site\cpd.html

# OWASP 依赖安全
start target\reports\dependency-check-report.html
```

## ❓ 常见问题

### Q: 脚本必须在 reports 目录运行吗？

A: 是的，所有 `test-*.bat` 和 `view-reports.bat` 都需要在 `reports` 目录运行。它们会自动切换到项目根目录执行 Maven 命令。

### Q: 为什么 test-all.bat 不包含 OWASP？

A: OWASP 依赖安全扫描耗时较长（5-10分钟），首次运行需要下载漏洞数据库。建议单独运行。

### Q: 如何只看覆盖率？

```cmd
cd reports
test-jacoco.bat
# 脚本会询问是否打开报告，输入 Y
```

### Q: 报告在哪里？

```cmd
cd reports
view-reports.bat
# 选择 1 查看覆盖率报告
```

### Q: 如何清理旧报告？

```cmd
mvn clean
```
