## 📊 质量徽章体系

### 1. 测试覆盖率徽章

#### 方案 A：使用 Codecov（推荐）

**优点**：
- ✅ 免费（开源项目）
- ✅ 自动生成徽章
- ✅ 支持多种 CI/CD
- ✅ 提供详细的覆盖率报告

**配置步骤**：

1. **注册 Codecov**
   - 访问 https://codecov.io
   - 使用 GitHub 账号登录
   - 添加 DLZ-DB 仓库

2. **添加 Codecov 配置**

创建 `.codecov.yml`：
```yaml
coverage:
  status:
    project:
      default:
        target: 15%  # 合理的目标
        threshold: 1%
    patch:
      default:
        target: 50%  # 新代码的目标

comment:
  layout: "reach,diff,flags,tree"
  behavior: default

ignore:
  - "**/*Test.java"
  - "**/test/**"
  - "**/dto/**"
  - "**/modal/**"
```

3. **在 GitHub Actions 中上传覆盖率**

`.github/workflows/ci.yml`：
```yaml
name: CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up JDK 8
      uses: actions/setup-java@v3
      with:
        java-version: '8'
        distribution: 'temurin'
    
    - name: Run tests with coverage
      run: mvn clean test jacoco:report
    
    - name: Upload coverage to Codecov
      uses: codecov/codecov-action@v3
      with:
        files: ./dlz-db-core/target/reports/jacoco/jacoco.xml
        flags: unittests
        name: codecov-dlz-db
```

4. **添加徽章到 README.md**

```markdown
[![codecov](https://codecov.io/gh/dingkui/dlz-db/branch/main/graph/badge.svg)](https://codecov.io/gh/dingkui/dlz-db)
```

### 2. 静态扫描徽章

#### 方案 A：使用 SonarCloud（推荐）

**优点**：
- ✅ 免费（开源项目）
- ✅ 全面的代码质量分析
- ✅ 自动生成多个徽章
- ✅ 支持多种语言

**配置步骤**：

1. **注册 SonarCloud**
   - 访问 https://sonarcloud.io
   - 使用 GitHub 账号登录
   - 导入 DLZ-DB 项目

2. **添加 SonarCloud 配置**

创建 `sonar-project.properties`：
```properties
sonar.projectKey=dingkui_dlz-db
sonar.organization=dingkui
sonar.projectName=DLZ-DB
sonar.projectVersion=7.0.0

# 源代码路径
sonar.sources=dlz-db-core/src/main/java
sonar.tests=dlz-db-core/src/test/java

# Java 版本
sonar.java.source=8
sonar.java.target=8

# 覆盖率报告
sonar.coverage.jacoco.xmlReportPaths=dlz-db-core/target/reports/jacoco/jacoco.xml

# 排除不需要扫描的文件
sonar.exclusions=**/dto/**,**/modal/**,**/annotation/**

# 质量门禁
sonar.qualitygate.wait=true
```

3. **在 GitHub Actions 中运行 SonarCloud**

```yaml
- name: SonarCloud Scan
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
    SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
  run: |
    mvn clean verify sonar:sonar \
      -Dsonar.projectKey=dingkui_dlz-db \
      -Dsonar.organization=dingkui \
      -Dsonar.host.url=https://sonarcloud.io
```

4. **添加徽章到 README.md**

```markdown

[![Bugs](https://sonarcloud.io/api/project_badges/measure?project=dingkui_dlz-db&metric=bugs)](https://sonarcloud.io/dashboard?id=dingkui_dlz-db)
[![Code Smells](https://sonarcloud.io/api/project_badges/measure?project=dingkui_dlz-db&metric=code_smells)](https://sonarcloud.io/dashboard?id=dingkui_dlz-db)
```

#### 方案 B：使用 CodeQL（GitHub 原生）

**优点**：
- ✅ GitHub 原生支持
- ✅ 免费
- ✅ 专注于安全漏洞

**配置**：

`.github/workflows/codeql.yml`：
```yaml
name: "CodeQL"

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]
  schedule:
    - cron: '0 0 * * 1'

jobs:
  analyze:
    name: Analyze
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v3

    - name: Initialize CodeQL
      uses: github/codeql-action/init@v2
      with:
        languages: java

    - name: Autobuild
      uses: github/codeql-action/autobuild@v2

    - name: Perform CodeQL Analysis
      uses: github/codeql-action/analyze@v2
```

### 3. 供应链安全徽章

#### 方案 A：使用 Snyk（推荐）

**优点**：

- ✅ 依赖漏洞扫描
- ✅ 自动生成徽章
- ✅ 自动 PR 修复

**配置步骤**：

1. **注册 Snyk**

2. **在 GitHub Actions 中运行 Snyk**

```yaml
- name: Run Snyk to check for vulnerabilities
  uses: snyk/actions/maven@master
  env:
    SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
  with:
    args: --severity-threshold=high
```

3. **添加徽章到 README.md**

```markdown
[![Known Vulnerabilities](https://snyk.io/test/github/dingkui/dlz-db/badge.svg)](https://snyk.io/test/github/dingkui/dlz-db)
```

#### 方案 B：使用 OWASP Dependency-Check（已配置）

**你已经配置了 OWASP Dependency-Check！** 现在需要：

1. **生成 JSON 报告**

在 `pom.xml` 中已配置：
```xml
<configuration>
    <format>ALL</format>  <!-- 包括 JSON -->
</configuration>
```

2. **在 CI 中运行**

```yaml
- name: OWASP Dependency Check
  run: mvn dependency-check:check
  
- name: Upload OWASP Report
  uses: actions/upload-artifact@v3
  with:
    name: dependency-check-report
    path: target/reports/dependency-check-report.html
```

3. **生成自定义徽章**

使用脚本解析 JSON 报告，生成徽章：

```bash
#!/bin/bash
# parse-owasp-report.sh

REPORT="target/reports/dependency-check-report.json"
HIGH_VULNS=$(jq '.dependencies[].vulnerabilities[] | select(.severity=="HIGH") | .name' $REPORT | wc -l)

if [ $HIGH_VULNS -eq 0 ]; then
  echo "![Security](https://img.shields.io/badge/security-passing-brightgreen)"
else
  echo "![Security](https://img.shields.io/badge/security-$HIGH_VULNS%20high-red)"
fi
```

## 🤖 完整的 CI/CD 配置

### GitHub Actions 完整配置

`.github/workflows/quality-check.yml`：

```yaml
name: Quality Check

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  quality:
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
      with:
        fetch-depth: 0  # SonarCloud 需要完整历史
    
    - name: Set up JDK 8
      uses: actions/setup-java@v3
      with:
        java-version: '8'
        distribution: 'temurin'
        cache: maven
    
    - name: Cache Maven packages
      uses: actions/cache@v3
      with:
        path: ~/.m2
        key: ${{ runner.os }}-m2-${{ hashFiles('**/pom.xml') }}
        restore-keys: ${{ runner.os }}-m2
    
    # 1. 运行测试和覆盖率
    - name: Run tests with coverage
      run: mvn clean test jacoco:report -pl dlz-db-core
    
    # 2. 上传覆盖率到 Codecov
    - name: Upload coverage to Codecov
      uses: codecov/codecov-action@v3
      with:
        files: ./dlz-db-core/target/reports/jacoco/jacoco.xml
        flags: unittests
        name: codecov-dlz-db
    
    # 3. 运行 SonarCloud 扫描
    - name: SonarCloud Scan
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
      run: |
        mvn sonar:sonar \
          -Dsonar.projectKey=dingkui_dlz-db \
          -Dsonar.organization=dingkui \
          -Dsonar.host.url=https://sonarcloud.io
    
    # 4. 运行 SpotBugs
    - name: Run SpotBugs
      run: mvn spotbugs:check
    
    # 5. 运行 PMD
    - name: Run PMD
      run: mvn pmd:check
    
    # 6. 运行 Checkstyle
    - name: Run Checkstyle
      run: mvn checkstyle:check
      continue-on-error: true  # Checkstyle 可能有很多警告
    
    # 7. 运行 OWASP Dependency Check
    - name: OWASP Dependency Check
      run: mvn dependency-check:check
      continue-on-error: true  # 可能有误报
    
    # 8. 运行 Snyk 安全扫描
    - name: Run Snyk
      uses: snyk/actions/maven@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --severity-threshold=high
      continue-on-error: true
    
    # 9. 上传报告
    - name: Upload reports
      uses: actions/upload-artifact@v3
      if: always()
      with:
        name: quality-reports
        path: |
          **/target/reports/**
          **/target/site/**
    
    # 10. 生成质量报告摘要
    - name: Generate quality summary
      run: |
        echo "## Quality Check Summary" >> $GITHUB_STEP_SUMMARY
        echo "" >> $GITHUB_STEP_SUMMARY
        echo "✅ Tests passed" >> $GITHUB_STEP_SUMMARY
        echo "📊 Coverage report uploaded to Codecov" >> $GITHUB_STEP_SUMMARY
        echo "🔍 Static analysis completed" >> $GITHUB_STEP_SUMMARY
        echo "🔒 Security scan completed" >> $GITHUB_STEP_SUMMARY
```

## 📊 质量指标自动化

### 1. 生成质量报告 JSON

创建脚本 `scripts/generate-quality-metrics.sh`：

```bash
#!/bin/bash

# 生成质量指标 JSON
cat > quality-metrics.json << EOF
{
  "timestamp": "$(date -u +%Y-%m-%dT%H:%M:%SZ)",
  "version": "7.0.0",
  "metrics": {
    "coverage": {
      "line": $(grep -oP 'line-rate="\K[^"]+' dlz-db-core/target/reports/jacoco/jacoco.xml | head -1),
      "branch": $(grep -oP 'branch-rate="\K[^"]+' dlz-db-core/target/reports/jacoco/jacoco.xml | head -1)
    },
    "tests": {
      "total": $(grep -oP 'tests="\K[^"]+' dlz-db-core/target/surefire-reports/*.xml | head -1),
      "passed": $(grep -oP 'tests="\K[^"]+' dlz-db-core/target/surefire-reports/*.xml | head -1),
      "failed": 0
    },
    "quality": {
      "bugs": 0,
      "vulnerabilities": 0,
      "code_smells": 23,
      "technical_debt": "45min"
    }
  }
}
EOF

echo "Quality metrics generated: quality-metrics.json"
```

### 2. 在 CI 中生成和上传

```yaml
- name: Generate quality metrics
  run: bash scripts/generate-quality-metrics.sh

- name: Upload quality metrics
  uses: actions/upload-artifact@v3
  with:
    name: quality-metrics
    path: quality-metrics.json
```
