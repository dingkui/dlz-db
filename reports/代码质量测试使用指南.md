# DLZ-DB 代码质量测试使用指南

## 📋 测试工具清单

| 工具 | 脚本文件 | 检测内容 | 报告位置 |
|------|---------|----------|----------|
| **JaCoCo** | `test-jacoco.bat` | 测试覆盖率 | `target/reports/jacoco-aggregate/index.html` |
| **SpotBugs** | `test-spotbugs.bat` | 静态分析 + 安全漏洞 | `target/reports/spotbugs/spotbugsXml.html` |
| **Checkstyle** | `test-checkstyle.bat` | 代码规范 | `target/site/checkstyle.html` |
| **PMD** | `test-pmd.bat` | 代码质量 + 重复代码 | `target/site/pmd.html` |
| **全部** | `test-all.bat` | 运行所有检查 | 多个位置 |

---

## 🚀 快速开始

### 方式一：运行所有检查（推荐）

```cmd
test-all.bat
```

**说明**：
- 运行所有 4 种质量检查
- 自动生成所有报告
- 完成后询问是否打开报告查看器

**耗时**：约 3-5 分钟

---

### 方式二：单独运行某个检查

#### 1. JaCoCo 测试覆盖率

```cmd
test-jacoco.bat
```

**检测内容**：
- 行覆盖率
- 分支覆盖率
- 方法覆盖率
- 类覆盖率

**报告位置**：
```
target/reports/jacoco-aggregate/index.html  （多模块聚合）
dlz-db-core/target/reports/jacoco/index.html  （Core模块）
```

**如何查看**：
1. 运行脚本后选择 Y 打开报告
2. 或手动打开 HTML 文件
3. 🟢 绿色 = 已覆盖，🔴 红色 = 未覆盖

---

#### 2. SpotBugs 静态分析

```cmd
test-spotbugs.bat
```

**检测内容**：
- Bug 模式（空指针、资源泄漏等）
- 安全漏洞（SQL注入、XSS等）
- FindSecBugs 安全检查

**报告位置**：
```
target/reports/spotbugs/spotbugsXml.html
```

**如何查看**：
1. 打开 HTML 报告
2. 查看问题优先级：
   - 🔴 High = 严重，必须修复
   - 🟡 Medium = 中等，建议修复
   - 🟢 Low = 轻微，可选修复

---

#### 3. Checkstyle 代码规范

```cmd
test-checkstyle.bat
```

**检测内容**：
- 命名规范
- 缩进和空格
- 导入顺序
- Javadoc 注释
- 代码结构

**标准**：Google Java Style Guide

**报告位置**：
```
target/site/checkstyle.html
target/reports/checkstyle/checkstyle-result.xml
```

**如何查看**：
1. 打开 HTML 报告
2. 查看违规列表
3. 点击文件名查看详细位置

---

#### 4. PMD 代码质量

```cmd
test-pmd.bat
```

**检测内容**：
- 圈复杂度
- 代码坏味道
- 未使用的变量
- 空的 catch 块
- 重复代码（CPD）

**报告位置**：
```
target/site/pmd.html  （PMD报告）
target/site/cpd.html  （重复代码报告）
```

**如何查看**：
1. 打开 PMD 报告查看代码质量问题
2. 打开 CPD 报告查看重复代码
3. 按优先级修复问题

---

## 💡 使用技巧

### 1. 并行运行（节省时间）

在不同的命令行窗口中同时运行：

```cmd
REM 窗口 1
test-jacoco.bat

REM 窗口 2
test-spotbugs.bat

REM 窗口 3
test-checkstyle.bat

REM 窗口 4
test-pmd.bat
```

---

### 2. 只运行 Maven 命令

如果不想用脚本，可以直接运行 Maven 命令：

```bash
# JaCoCo
mvn test jacoco:report

# SpotBugs
mvn compile spotbugs:check

# Checkstyle
mvn checkstyle:check

# PMD
mvn pmd:check pmd:cpd-check

# 全部
mvn clean test jacoco:report spotbugs:check checkstyle:check pmd:check
```

---

### 3. 在 IDE 中查看

**IntelliJ IDEA**：
- 覆盖率：右键测试类 → "Run with Coverage"
- SpotBugs：安装 SpotBugs 插件
- Checkstyle：安装 Checkstyle 插件
- PMD：安装 PMD 插件

**Eclipse**：
- 覆盖率：安装 EclEmma 插件
- 其他：安装对应插件

## ❓ 常见问题

### Q1: 首次运行很慢？

**A**: 正常现象，Maven 需要下载插件。后续运行会快很多。

---

### Q2: 报告打不开？

**A**: 确保先运行了对应的测试脚本生成报告。

---

### Q3: 如何修改质量标准？

**A**: 编辑 `pom.xml` 中的插件配置：

```xml
<properties>
    <!-- JaCoCo 覆盖率阈值 -->
    <jacoco.line.coverage>0.70</jacoco.line.coverage>
    <jacoco.branch.coverage>0.60</jacoco.branch.coverage>
</properties>
```

---

### Q4: 如何忽略某些警告？

**A**: 
- **SpotBugs**: 使用 `@SuppressFBWarnings` 注解
- **Checkstyle**: 使用 `@SuppressWarnings("checkstyle:...")`
- **PMD**: 使用 `@SuppressWarnings("PMD....")`

---

### Q5: 报告在哪里？

**A**: 运行 `view-reports.bat` 查看所有报告位置。
