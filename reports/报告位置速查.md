### 3. 静态分析 (SpotBugs)

```
target/spotbugs/spotbugsXml.html
```

**快速打开**：
```cmd
# Windows
start target\spotbugs\spotbugsXml.html

# Linux/Mac
open target/spotbugs/spotbugsXml.html
```

**问题级别**：
- 🔴 **High**: 严重问题，必须修复
- 🟡 **Medium**: 中等问题，建议修复
- 🟢 **Low**: 轻微问题，可选修复
- 🔒 **Security**: 安全漏洞（FindSecBugs）

**常见问题**：
- `NP_NULL_ON_SOME_PATH` - 可能的空指针
- `RCN_REDUNDANT_NULLCHECK_OF_NONNULL_VALUE` - 冗余的空检查
- `SQL_INJECTION` - SQL 注入风险
- `XSS_REQUEST_PARAMETER_TO_SERVLET_WRITER` - XSS 漏洞

## 🎯 常用命令

### 生成报告

```bash
# 快速检查（测试 + 覆盖率 + SpotBugs）
quick-check.bat         # Windows
./quick-check.sh        # Linux/Mac

# 完整检查（所有工具）
run-quality-checks.bat  # Windows
./run-quality-checks.sh # Linux/Mac
```

### 查看报告

```bash
# 交互式报告查看器
view-reports.bat        # Windows
./view-reports.sh       # Linux/Mac
```

### 单独生成某个报告

```bash
# 只生成覆盖率报告
mvn test jacoco:report

# 只运行 SpotBugs
mvn compile spotbugs:check

# 只运行 Checkstyle
mvn checkstyle:check

# 只运行 PMD
mvn pmd:check

# 只检测重复代码
mvn pmd:cpd-check

# 生成综合站点
mvn site
```

## 💡 使用技巧

### 1. 查看覆盖率详情

打开 `target/site/jacoco-aggregate/index.html` 后：
1. 点击包名查看包级别覆盖率
2. 点击类名查看类级别覆盖率
3. 点击方法名查看方法级别覆盖率
4. 查看源代码，红色表示未覆盖

### 2. 理解 SpotBugs 报告

- 点击问题类型查看详细说明
- 查看 "Bug Pattern" 了解问题原因
- 参考 "How to fix" 了解修复方法

### 3. 过滤 Checkstyle 违规

- 按严重程度排序
- 优先修复 "Error" 级别
- "Warning" 级别可以逐步改进

### 4. 分析 PMD 报告

- 关注高优先级问题
- 查看圈复杂度高的方法
- 重构过长的方法和类

### 5. 处理重复代码

- 查看 CPD 报告找到重复代码
- 提取公共方法
- 使用继承或组合消除重复

## ❓ 常见问题

### Q: 报告文件不存在？

**A**: 先运行质量检查：
```bash
quick-check.bat         # 快速检查
run-quality-checks.bat  # 完整检查
```

### Q: 覆盖率报告是空的？

**A**: 确保运行了测试：
```bash
mvn clean test jacoco:report
```

### Q: 如何在 IDE 中查看覆盖率？

**A**: 
- **IntelliJ IDEA**: 右键测试类 → "Run with Coverage"
- **Eclipse**: 安装 EclEmma 插件
- 或直接在浏览器中打开 HTML 报告

### Q: SpotBugs 报告太多问题？

**A**: 
1. 优先修复 High 级别问题
2. 使用 `@SuppressFBWarnings` 抑制误报
3. 逐步改进，不要一次性修复所有问题

### Q: 如何自定义报告位置？

**A**: 在 `pom.xml` 中配置插件的 `outputDirectory` 参数
