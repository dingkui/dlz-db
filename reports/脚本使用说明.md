# 质量测试脚本使用说明

## 📂 目录结构

```
dlz-db/
├── quality-test.bat          # 项目根目录入口（可选）
│
└── reports/                  # 测试脚本目录 ⭐
    ├── test-all.bat          # 运行所有检查
    ├── test-jacoco.bat       # JaCoCo 覆盖率
    ├── test-spotbugs.bat     # SpotBugs 静态分析
    ├── test-checkstyle.bat   # Checkstyle 代码规范
    ├── test-pmd.bat          # PMD 代码质量
    ├── test-owasp.bat        # OWASP 依赖安全
    ├── view-reports.bat      # 报告查看器
    │
    ├── README.md             # 快速开始
    ├── 如何运行测试.md       # 运行指南
    └── 脚本使用说明.md       # 本文档
```

---

## 🚀 运行方式

### ⚠️ 重要：所有脚本必须在 `reports` 目录下运行

```cmd
# 1. 进入 reports 目录
cd reports

# 2. 运行脚本
test-all.bat
```

### 为什么必须在 reports 目录？

脚本内部使用 `cd ..` 切换到项目根目录执行 Maven 命令，然后用 `cd reports` 返回。如果在其他目录运行会导致路径错误。

## 📋 脚本详解

### 1. test-all.bat - 完整检查

**功能**：运行所有质量检查（不含 OWASP）

**运行**：
```cmd
cd reports
test-all.bat
```

**执行步骤**：
1. 切换到项目根目录 (`cd ..`)
2. 编译项目 (`mvn clean compile -DskipTests`)
3. 运行测试 + JaCoCo (`mvn test jacoco:report`)
4. SpotBugs 分析 (`mvn spotbugs:check`)
5. Checkstyle 检查 (`mvn checkstyle:check`)
6. PMD 分析 (`mvn pmd:check pmd:cpd-check`)
7. 返回 reports 目录 (`cd reports`)
8. 询问是否打开报告查看器

**耗时**：3-5 分钟

---

### 2. test-jacoco.bat - 测试覆盖率

**功能**：运行单元测试并生成覆盖率报告

**运行**：
```cmd
cd reports
test-jacoco.bat
```

**执行步骤**：
1. 切换到项目根目录
2. 运行 `mvn clean test jacoco:report`
3. 返回 reports 目录
4. 询问是否打开报告

**报告位置**：
- `target\reports\jacoco-aggregate\index.html`（聚合报告）
- `dlz-db-core\target\reports\jacoco\index.html`（Core 模块）

**耗时**：1-2 分钟

---

4. 询问是否打开报告

**报告位置**：

**耗时**：1-2 分钟

### 6. test-owasp.bat - 依赖安全

**功能**：OWASP 依赖安全扫描

**运行**：
```cmd

```

**执行步骤**：
1. 切换到项目根目录
2. 运行 `mvn dependency-check:check`
3. 返回 reports 目录
4. 询问是否打开报告

**报告位置**：
- `target\reports\dependency-check-report.html`

**耗时**：5-10 分钟（首次运行需下载漏洞数据库，更长）

**注意**：由于耗时较长，`test-all.bat` 不包含此检查。

---

### 7. view-reports.bat - 报告查看器

**功能**：交互式报告查看器

**运行**：
```cmd
cd reports
view-reports.bat
```

**执行步骤**：
1. 切换到项目根目录
2. 检查所有报告是否存在
3. 显示交互式菜单
4. 根据用户选择打开报告
5. 返回 reports 目录

**菜单选项**：
- 1: JaCoCo 覆盖率报告（聚合）
- 2: dlz-db-core 覆盖率
- 3: dlz-db-spring-boot-starter 覆盖率
- 4: dlz-db-solon-plugin 覆盖率
- 5: SpotBugs 静态分析
- 6: Checkstyle 代码规范
- 7: PMD 代码质量
- 8: CPD 重复代码
- 9: 测试报告目录
- 0: 打开所有报告

## ⚙️ 脚本工作原理

所有脚本遵循相同的模式：

```batch
@echo off
chcp 65001 >nul              # 支持中文显示

# 1. 切换到项目根目录
cd ..

# 2. 运行 Maven 命令
call mvn <goal>

# 3. 返回 reports 目录
cd reports

# 4. 询问是否打开报告
set /p open="是否打开报告？(Y/N): "
if /i "%open%"=="Y" (
    start "" "<report-path>"
)

pause
```

## ❓ 常见问题

### Q: 为什么脚本不能在项目根目录运行？

A: 脚本设计为在 `reports` 目录运行，使用相对路径 `cd ..` 切换到根目录。如果在根目录运行，`cd ..` 会跳到父目录，导致路径错误。

### Q: 如何从项目根目录快速运行？

A: 使用入口脚本：
```cmd
quality-test.bat
```
它会自动进入 `reports` 目录并运行 `test-all.bat`。

### Q: 脚本运行失败怎么办？

A: 检查以下几点：
1. 确保在 `reports` 目录运行
2. 确保 Maven 已安装并配置
3. 确保项目可以正常编译
4. 查看错误信息，可能是某个插件配置问题

### Q: 如何清理旧报告？

```cmd
mvn clean
```

### Q: 报告查看器显示"未找到报告"？

A: 需要先运行测试生成报告：
```cmd
cd reports
test-all.bat
```
